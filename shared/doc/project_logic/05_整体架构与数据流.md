## 3. 完整数据流程

### 3.1 编辑自定义标题流程

```mermaid
sequenceDiagram
    participant User as 用户
    participant DOM as 编辑器 DOM
    participant Observer as editorObserver
    participant Scanner as scanEditorFlashcards
    participant Button as 编辑按钮
    participant Dialog as 对话框
    participant API as api.ts
    participant SiYuan as 思源内核

    User->>DOM: 打开包含闪卡的文档
    DOM->>Observer: 触发 MutationObserver
    Observer->>Scanner: 调用扫描函数
    Scanner->>DOM: 查询 [custom-riff-decks] 元素
    Scanner->>Button: 为每个闪卡创建按钮
    Button->>DOM: 添加到 protyle-attr
    
    User->>Button: 点击编辑按钮
    Button->>Dialog: 打开对话框
    Dialog->>API: getBlockAttrs blockId
    API->>SiYuan: /api/attr/getBlockAttrs
    SiYuan-->>API: 返回属性
    API-->>Dialog: custom-riff-title 值
    Dialog->>Dialog: 显示当前标题
    
    User->>Dialog: 输入新标题并保存
    Dialog->>API: setBlockAttrs blockId, attrs
    API->>SiYuan: /api/attr/setBlockAttrs
    SiYuan-->>API: 保存成功
    API-->>Dialog: 操作完成
    Dialog->>User: 显示成功提示
```

### 3.2 复习界面标题替换流程

```mermaid
sequenceDiagram
    participant User as 用户
    participant SiYuan as 思源笔记
    participant DOM as 复习界面 DOM
    participant Observer as reviewObserver
    participant Replacer as replaceFlashcardTitle
    participant API as api.ts
    participant Kernel as 思源内核

    User->>SiYuan: 打开闪卡复习界面
    SiYuan->>DOM: 渲染复习界面
    DOM->>Observer: 触发 MutationObserver
    Observer->>Observer: 检查是否在复习界面内
    Observer->>Replacer: 调用标题替换函数
    
    Replacer->>DOM: 获取闪卡元素
    Replacer->>DOM: 获取 custom-riff-title 属性
    
    alt 有自定义标题
        Replacer->>DOM: 直接替换标题文本
    else 无自定义标题
        Replacer->>API: getBlockByID headingBlockId
        API->>Kernel: SQL 查询
        Kernel-->>API: 返回块信息
        API-->>Replacer: 原标题内容
        Replacer->>DOM: 恢复原标题
    end
    
    Replacer->>DOM: 设置 data-title-replaced 标记
```

## 4. 关键技术点

### 4.1 MutationObserver 监听策略

```mermaid
flowchart TD
    A[MutationObserver] --> B{监听类型}
    
    B --> C[childList]
    B --> D[attributes]
    
    C --> E[新增节点检测]
    E --> F{节点类型判断}
    F -->|protyle-wysiwyg| G[编辑器视图变化]
    F -->|custom-riff-decks| H[闪卡元素]
    F -->|复习界面内| I[复习界面变化]
    
    D --> J{属性类型}
    J -->|custom-riff-decks| K[闪卡属性变化]
    J -->|data-node-id| L[块 ID 变化]
    J -->|custom-riff-title| M[标题属性变化]
    J -->|class| N[标签切换检测]
```

### 4.2 界面判断逻辑

```mermaid
flowchart TD
    A[元素检测] --> B{向上查找父元素}
    B --> C{找到 data-key=dialog-opencard?}
    C -->|是| D[复习界面]
    C -->|否| E{找到 data-loading=finished?}
    E -->|是| F[编辑器界面]
    E -->|否| G[其他界面]
```

### 4.3 标题替换防重复机制

```mermaid
flowchart TD
    A[检测到闪卡] --> B{已设置 data-title-replaced?}
    B -->|是| C[跳过处理]
    B -->|否| D[执行标题替换]
    D --> E[设置 data-title-replaced=true]
    E --> F[处理完成]
    
    G[属性变化] --> H{custom-riff-title 变化?}
    H -->|是| I[移除 data-title-replaced]
    I --> D
    H -->|否| J[忽略]
```

## 7. 扩展建议

### 7.1 当前架构优势
- 模块化设计，功能独立
- 使用 MutationObserver 实现无侵入式监听
- API 层封装良好，易于维护

### 7.2 潜在改进点
1. **状态管理**：可引入 Pinia 进行全局状态管理
2. **配置持久化**：添加插件配置存储功能
3. **国际化完善**：补充更多语言支持
4. **错误边界**：添加更完善的错误处理机制
